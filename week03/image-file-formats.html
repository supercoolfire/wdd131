    Image File Formats

    Notes Overview

    I have worked with various image formats, some of which are better suited for the web. Responsive image design aims to reduce page weight and deliver optimized images for different screen sizes. This activity explores the SVG (Scalable Vector Graphics) and WebP formats, which offer better compression for faster page loads.

    Prepare

    • Review: Video ▶️ Image File Types for the Web – [ 1:54 minutes ]
      • jpeg: compression good for photographs
      • gif: limited color palette, good for simple animations
      • png: lossless compression, good for graphics with transparency
      • webp: modern format with better compression, supports transparency and animation, supports animation, vectors, shapes, text, logo
      • heic: modern format for iPhone photos, offers better compression than JPEG. Not supported on the web
    • Reference: 📑WebP image format – developers.google.com
      • Lossless Compression: using sophisticated algorithms that replace redundancy. 26% smaller to PNG
      • Lossy Compression: using approximations and parital data to represent the original media source. 25-34% smaller to JPEG
      • Transparency Support: property representing how much of the background color is visible through the image. 22% additional bytes
    • Reference: 📑 Image file type and format guide – MDN
      • SVG: Vector image ideal for ui, icons, diagrams etc.Support: Chrome, Edge, Firefox, IE, Opera, Safari
      • WebP: Excellent choice better than PNG or JPEG, supoprt: Chrome, Edge, Firefox, Opera, Safary .
